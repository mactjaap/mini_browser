mini_browser — tiny text-mode web for the WHY2025 badge

mini_browser is a small, fast, text-centric browser built for the WHY2025 badge.
It fetches pages with libcurl, strips them down to text, collects links, and renders everything with a 5×7 bitmap font via SDL.

What it can do

  • Follow links — Every <a href> is collected and numbered like [12]. Use Tab / Shift+Tab to select, Enter to open.
  • Type URLs directly — Type in the address bar; Enter to go. Bare hosts auto-expand (e.g. example.org → https://example.org).
  • Google search box — A minimal search control is baked in for Google’s homepage.
  • Redirects — Handles 3xx redirects.
  • Relative URLs — Resolves paths like /foo, ../bar, //host/…
  • Entity decoding — Decodes &amp; &lt; &gt; &quot; '
  • Keyboard-only — Optimized for the badge keyboard.

Keys & shortcuts

  • Type: edit URL/search field
  • Enter: open typed URL or selected link
  • Backspace: delete in the address bar
  • Tab / Shift+Tab: next/previous link
  • ↑ / k: scroll up
  • ↓ / j: scroll down
  • PageUp / PageDown: faster scrolling
  • Home: jump to top
  • Ctrl+R: reload
  • Ctrl+Q / Esc: quit
  • Ctrl+H: open https://httpbin.org/html
  • Ctrl+E: open https://badge.why2025.org/

How it works

  1. Fetch — libcurl downloads up to MAX_BYTES (16 KB by default) with timeouts.
  2. Parse — A tiny HTML walker:
    • Skips <head>, <script>, <style>
    • Emits text, collapses whitespace, decodes entities
    • Captures <a href> targets, appends numbered markers like [7]
    • Resolves relative URLs against the current page
  3. Wrap — Text is wrapped to display width (monospace 5×7 font).
  4. Render — SDL draws the URL bar and text content; navigation by keyboard.

Google search (minimal)

On www.google.com, the browser exposes a minimal search input:

  • Press Enter while the search box is selected, type your query, Enter again
  • Navigates to https://www.google.com/search?q=…
  • You can also type https://www.google.com/search?q=your+query directly in the bar

Tip: If you mainly search, set the start URL to https://www.google.com/search?q= and just type your terms after q=.

Known limits

  • No CSS/JS/Images — text only, scripts/styles skipped.
  • ASCII-first — Non-ASCII glyphs simplified, unknowns → ?
  • No cookies/sessions — no persistent jar.
  • Body truncated to MAX_BYTES (default 16 KB).
